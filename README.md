# Taýpa Telegram Post Bot

Бот принимает фотографию, определяет тип вещи, придумывает название принта через Gemini API, предлагает выбрать размеры, спрашивает цену, показывает предпросмотр и публикует пост в Telegram-канале сразу или в выбранное время. Отдельный режим создает по готовому макету реалистичные фотографии одежды на разных людях.

## Возможности

- Доступ только для владельца по Telegram ID.
- Автоматическое определение вещи: футболка, худи, свитшот, лонгслив, зип-худи,
  куртка, кепка или шопер.
- Короткое подходящее название принта в кавычках.
- Два хэштега: один по типу вещи, второй по тематике принта.
- Выбор размеров кнопками: S-XXL, XS-4XL, XS-3XL или свой вариант.
- Запрос цены после анализа фотографии.
- Быстрый выбор времени или ввод точной даты.
- Предпросмотр перед подтверждением.
- Кнопка «Написать» открывает личный чат и заранее вставляет название товара.
- Изменение текста, цены и времени до публикации.
- Полное управление очередью: предпросмотр, редактирование, публикация,
  копирование и отмена.
- Постоянное хранение очереди, шаблона и пресетов в Neon PostgreSQL на Render.
- Автоматический резервный режим SQLite для запуска на Windows.
- До 5 автоматических попыток при временной ошибке Telegram.
- Редактируемый шаблон подписи.
- Готовые пресеты размеров и цен с возможностью добавлять свои.
- Одна экономная генерация по готовому макету, следующий кадр только по кнопке.
- Каждая генерация обязательно использует один подходящий референс из каталога.
- Референс управляет позой, кадрированием и фоном, а исходное фото остается единственным источником цвета, ткани и принта.
- Свободная генерация без референса заблокирована, чтобы бот не придумывал новую сцену и новый товар.
- Автоматическое определение передней или задней стороны, цвета, ткани,
  размера и положения принта.
- Отдельная физика DTF для одежды и кепок с учетом изгиба, шва панели и строчек.
- Вертикальный формат 4:5 с безопасными полями для разных устройств.
- Выбор готового кадра и переход к созданию поста одной кнопкой.
- Каталог референсов в Neon с локальными копиями, тегами Gemini и историей
  использования. Приоритет получают близкие кадры без толпы и с правильной стороной принта.
- Команда `/check` показывает ID последнего референса, факт передачи в Gemini и статус генерации.
- Фоновый импорт 63 стартовых ссылок без остановки Telegram-бота.
- Загрузка новых TXT-списков референсов прямо через Telegram.
- Анализ макета имеет приоритет над фоновой разметкой референсов.
- Перед анализом изображение автоматически приводится к поддерживаемому JPEG.
- При временном сбое Gemini бот делает до трех попыток и сохраняет макет для
  кнопки `Повторить анализ`.
- Координаты Gemini автоматически распознаются в шкале 0-1, процентах или
  пикселях, включая смешанный формат для вещи и принта.
- Невозможная геометрия и низкая уверенность запускают повторный бесплатный
  анализ и блокируют платную генерацию.

## Быстрый запуск на Windows

### 1. Создайте бота

1. Откройте в Telegram `@BotFather`.
2. Отправьте команду `/newbot`.
3. Укажите название и username нового бота.
4. Скопируйте выданный токен. Он нужен для `TELEGRAM_BOT_TOKEN`.

### 2. Подготовьте канал

1. Откройте настройки канала.
2. Перейдите в раздел администраторов.
3. Добавьте созданного бота.
4. Включите ему право публикации сообщений.
5. Если канал публичный, скопируйте его username вместе со знаком `@`. Он нужен для `CHANNEL_ID`.

### 3. Узнайте ADMIN_TELEGRAM_ID

Это ваш личный цифровой ID. Это не номер телефона и не username.

1. Откройте в Telegram бота `@userinfobot`.
2. Нажмите `Start`.
3. Найдите строку `Id`, например `Id: 726543210`.
4. В `.env` запишите только цифры: `ADMIN_TELEGRAM_ID=726543210`.

Этот ID нужен, чтобы управлять ботом могли только вы.

### 4. Укажите CHANNEL_ID

Если канал публичный и его ссылка выглядит как `t.me/taypa_print`, используйте:

```env
CHANNEL_ID=@taypa_print
```

Если канал закрытый и username отсутствует, нужен цифровой ID вида `-1001234567890`:

1. Добавьте своего бота администратором канала.
2. Опубликуйте в канале любое тестовое сообщение.
3. Перешлите его боту `@RawDataBot`.
4. В ответе найдите блок `chat` и поле `id`.
5. Скопируйте полное число, начинающееся с `-100`.

### 5. Укажите CONTACT_USERNAME

Это ваш личный username, на который ведет кнопка «Написать».

1. Откройте Telegram, затем `Настройки` и `Имя пользователя`.
2. Если username еще нет, создайте его.
3. Укажите его без знака `@`.

Например, для ссылки `t.me/taypa_manager` используйте:

```env
CONTACT_USERNAME=taypa_manager
```

### 6. Создайте ключ Gemini API

1. Откройте `https://aistudio.google.com/app/apikey`.
2. Войдите в Google-аккаунт.
3. Нажмите `Create API key`.
4. Скопируйте ключ. Он нужен для `GEMINI_API_KEY`.
5. Никому не отправляйте этот ключ.

Обычный анализ фотографий может работать на бесплатном тарифе Gemini API.
Генерация новых фотографий через `gemini-3.1-flash-lite-image` требует включенного
биллинга Google. Бот запускается и публикует обычные посты даже без биллинга,
но кнопка `Фото на модели` сообщит, что платный доступ не включен.

### 7. Установите и запустите

1. Установите Python 3.11 или новее с сайта `python.org`. При установке включите пункт `Add Python to PATH`.
2. Запустите `setup.bat` и дождитесь окончания установки.
3. Откройте созданный файл `.env` в Блокноте.
4. Замените примеры своими данными и сохраните файл.
5. Запустите `run.bat`. Не закрывайте это окно, пока бот должен работать.
6. Откройте бота в Telegram и отправьте `/start`.
7. Отправьте `/check`, чтобы проверить доступ к каналу.

Если ранее появлялась ошибка `неизвестный часовой пояс Asia/Ashgabat`, снова запустите обновленный `setup.bat`. Он установит базу часовых поясов для Windows. В боте также предусмотрен запасной режим UTC+5.

## Настройка .env

```env
TELEGRAM_BOT_TOKEN=токен_от_BotFather
GEMINI_API_KEY=ключ_Gemini_из_Google_AI_Studio
ADMIN_TELEGRAM_ID=ваш_цифровой_Telegram_ID
CHANNEL_ID=@имя_канала
CONTACT_USERNAME=ваш_username_без_знака_собака
TIMEZONE=Asia/Ashgabat
GEMINI_MODEL=gemini-3.1-flash-lite
GEMINI_IMAGE_MODEL=gemini-3.1-flash-lite-image
GEMINI_IMAGE_SIZE=1K
MOCKUP_VARIANTS=1
REFERENCE_IMPORT_DELAY_SECONDS=5
REFERENCE_IDLE_INTERVAL_SECONDS=300
REFERENCE_MAX_ATTEMPTS=5
REFERENCE_MIN_POOL_SIZE=20
REFERENCE_ANALYSIS_TIMEOUT_SECONDS=90
MOCKUP_ANALYSIS_TIMEOUT_SECONDS=150
BUTTON_TEXT=Написать
COPY_LANGUAGE=ru
# Необязательно на Windows. На Render укажите строку подключения Neon.
DATABASE_URL=
DATABASE_PATH=data/posts.sqlite3
CAPTION_TEMPLATE_PATH=caption_template.txt
```

Для закрытого канала в `CHANNEL_ID` укажите числовой ID вида `-1001234567890`.

Если в перенесенном из старой версии `.env` осталась модель
`gemini-2.5-flash`, бот автоматически заменит ее на актуальную модель.

## Команды

- `/start` - открыть главное меню.
- `/queue` - показать запланированные посты.
- `/template` - показать шаблон подписи.
- `/settemplate` - изменить шаблон прямо в Telegram.
- `/presets` - показать, добавить или удалить готовые пресеты.
- `/model` - создать реалистичные фотографии из готового макета.
- `/references` - посмотреть и пополнить каталог референсов.
- `/check` - проверить доступ бота к каналу.
- `/cancel` - отменить создание текущего черновика.

## Как опубликовать пост

1. Отправьте боту фотографию.
2. Подождите, пока он определит вещь и создаст название принта.
3. Выберите готовый пресет или ручной ввод.
4. При ручном вводе выберите размеры и отправьте цену. Если написать только
   `250`, бот поставит `250 манат`.
5. Выберите быструю дату или нажмите «Другая дата и время».
6. Проверьте фотографию, подпись, цену, размеры, время и кнопку в предпросмотре.
7. При необходимости измените название, размеры, цену или время.
8. Нажмите «Подтвердить».
9. Через `/queue` можно посмотреть, изменить, опубликовать, скопировать или
   отменить будущую публикацию.

## Как создать фотографии по макету

1. Нажмите `Фото на модели` или отправьте `/model`.
2. Отправьте фото или макет одежды с уже размещенным принтом. Для максимальной
   точности отправляйте PNG, JPEG или WEBP как файл, а не как сжатую фотографию.
3. Бот бесплатно определит вид изделия, сторону, цвет, материал, конструкцию,
   крой, аудиторию, возрастную категорию и настроение. Размер и положение принта
   рассчитываются из двух найденных областей: рабочей части изделия и полного
   контура принта.
   Перед отправкой в Gemini файл автоматически поворачивается по EXIF,
   приводится к RGB и получает стандартный тип `image/jpeg`.
4. Проверьте карточку анализа. На этом шаге платная генерация еще не запускается.
   Уверенность приводится к единой шкале 0-100. Перед показом карточки бот
   проверяет, что принт находится внутри рабочей части вещи, а размер, отступы и
   центр согласованы между собой.
5. Отдельный PNG не обязателен. При наличии исходника можно нажать `Добавить PNG принта - необязательно` и отправить его как файл.
   Бот проверит прозрачность, найдет реальную видимую область и уточнит аудиторию,
   настроение и тему по оригинальному изображению.
6. Нажмите `Все верно - создать фото`. Только эта кнопка запускает одну платную
   генерацию 4:5.
7. Нажмите `Создать пост из этого` под подходящим вариантом или нажмите
   `Еще 1 вариант`.

Анализ хранится 48 часов в Neon вместе с Telegram file ID исходного макета и
PNG. Если Render перезапустится после показа карточки, кнопка подтверждения сможет
восстановить данные без повторного анализа.

Если Gemini временно достиг лимита, вернул неполный JSON или не ответил, бот не
объявляет макет нечетким. Он повторяет запрос, приостанавливает фоновую разметку
референсов и показывает настоящую причину. Кнопка `Повторить анализ` использует
тот же Telegram file ID, поэтому повторно отправлять изображение не нужно даже
после перезапуска Render в пределах 48 часов.

Gemini иногда возвращает уверенность в шкале 0-1, а границы принта в пикселях,
даже если границы вещи указаны в процентах. Бот нормализует каждую область
отдельно. Если результат все равно не проходит проверку, кнопка платной
генерации не показывается. Анализы, созданные старой версией без этой проверки,
нужно один раз повторить после обновления.

Относительные координаты сохраняются независимо от разрешения исходника:

```text
ширина принта / ширина рабочей части изделия
высота принта / высота рабочей части изделия
отступ слева / ширина рабочей части изделия
отступ сверху / высота рабочей части изделия
центр принта X / ширина рабочей части изделия
```

Эти значения станут входными данными для алгоритмического переноса принта на
человека на следующем этапе. Коэффициенты для среднего и очень крупного
телосложения будут применяться уже к целевой фотографии, а не искажать измерения
исходного макета.

Исходный принт передается модели как зафиксированный элемент. В запросе
запрещено менять буквы, орнаменты, пропорции и положение изображения. Прозрачная
область вокруг принта остается тканью, поэтому бот не должен придумывать
прямоугольную подложку. Цвет подстраивается только под свет реального кадра.

На одежде DTF остается тонким непрозрачным слоем на поверхности и повторяет
изгибы и складки. На кепке DTF ложится по округлой панели. Центральный шов,
панельные соединения и строчки козырька остаются слегка видимыми. Если принт
пересекает шов, появляется только небольшая естественная деформация. Бот не
превращает DTF в вышивку или нашивку.

Кадр создается как обычная фотография на телефон во время действия. Человек
может сидеть, идти, выбирать покупки, листать журнал, садиться в машину или
держать кофе. Допускаются естественный ракурс, обычный фон, легкий шум, мягкая
нерезкость движения и небольшая погрешность экспозиции. Запрещены студийная
идеальность, пластиковая кожа, рекламная стойка и искусственное размытие фона.
Генеративная модель все равно может изредка исказить очень мелкий текст, поэтому
перед публикацией обязательно проверьте результат.

Стандартный размер `1K` создает вертикальное изображение примерно 928 на 1152
пикселя. Это формат 4:5, подходящий для Telegram. Полное изделие и принт
удерживаются минимум в 8 процентах от краев. Руки, ноги и фон можно кадрировать
свободнее, поэтому фото выглядит живее, но важные детали не обрезаются.

Gemini не обучается на удачных результатах этого бота и не запоминает прошлые
промпты сам. Поэтому 63 стартовых референса теперь хранятся в отдельном каталоге
Neon вместе с тегами и историей использования. На Этапе 1 каталог подготавливает
данные и умеет выбирать подходящий кадр. Подключение выбранного изображения к
созданию чистой сцены выполняется на следующем этапе. Текущий генератор до этого
продолжает использовать встроенные бытовые сценарии.

## Каталог референсов

При первом запуске бот добавляет 63 ссылки из `reference_sources.txt` в очередь.
Импорт идет в фоне по одному изображению и не задерживает запуск Telegram.
Для каждого доступного изображения бот:

1. Сохраняет оптимизированную копию и миниатюру в Neon.
2. Один раз просит Gemini определить одежду, пол, настроение, действие, локацию,
   ракурс, кадрирование, освещение, сезон и видимость зоны нанесения.
3. Отключает коллажи, рисунки, детские фото и кадры, где одежда плохо видна.
4. Сохраняет счетчик и историю использования.

Команда `/references` отдельно показывает очередь, текущую обработку и ссылки,
которые ждут повторной попытки. Также выводятся время следующей попытки и общая
причина ожидания. Кнопка `Продолжить сейчас` немедленно возвращает отложенные и
зависшие задачи в очередь. Кнопка `Добавить список ссылок` принимает TXT-файл
или одно сообщение с новыми ссылками. Дубликаты не добавляются.

Каталог не использует аккаунт Pinterest, прокси, обход капчи или подмену
ограничений. Он медленно обрабатывает только явно добавленные ссылки, кеширует
результат и соблюдает `Retry-After`. Если Pinterest временно ограничит запросы,
бот поставит ссылку на повторную попытку. Сохраненная копия остается доступной,
даже если исходная ссылка позже исчезнет.

После выбора референс получает паузу 30 дней. Алгоритм предпочитает наименее
использованные варианты и оценивает совпадение по одежде, полу, настроению,
сезону и видимости места для принта. Целевой запас задается через
`REFERENCE_MIN_POOL_SIZE`, по умолчанию 20. Автоматический поиск новых чужих
пинов не выполняется. Для пополнения нужно отправить новый список разрешенных
ссылок.

Активный черновик поста хранится в базе 48 часов. Если Render перезапустит бота
между созданием текста и нажатием кнопки, бот восстановит размеры, цену и другие
данные и продолжит работу.

По опубликованным тарифам Google модель `gemini-3.1-flash-lite-image` стоит около
0.0336 доллара за одно изображение 1K без учета небольшого входного запроса.
Актуальная цена:
`https://ai.google.dev/gemini-api/docs/pricing`.

Дополнительные настройки:

- `GEMINI_IMAGE_SIZE=1K` - стандартный и наиболее экономный размер.
- Для модели Lite доступны только изображения 1K.
- `MOCKUP_VARIANTS=1` до `4` - количество кадров в одной серии.
- `REFERENCE_IMPORT_DELAY_SECONDS=5` - пауза между ссылками.
- `REFERENCE_IDLE_INTERVAL_SECONDS=300` - проверка очереди, когда она пуста.
- `REFERENCE_MAX_ATTEMPTS=5` - число повторов временно недоступной ссылки.
- `REFERENCE_MIN_POOL_SIZE=20` - целевой запас на используемую категорию.
- `REFERENCE_ANALYSIS_TIMEOUT_SECONDS=90` - предел ожидания ответа Gemini.
- `MOCKUP_ANALYSIS_TIMEOUT_SECONDS=150` - общий предел интерактивного анализа с
  учетом повторных попыток.

Чтобы пост вышел в заданное время, программа должна быть запущена. Для
круглосуточной работы используйте Render или VPS.

Никому не отправляйте файл `.env`, токен Telegram и ключ Gemini. Если секрет попал к постороннему, сразу замените его.

## Шаблон подписи

Файл `caption_template.txt` можно менять вручную или командой `/settemplate`. Доступные русские поля:

- `{Тип товара}` - футболка, худи, свитшот, лонгслив, зип-худи, куртка, кепка
  или шопер.
- `{Название принта}` - название, созданное по изображению.
- `{Короткое описание, передающее настроение принта}` - короткая фраза для товара.
- `{Размеры}` - выбранные размеры.
- `{Цена}` - цена, которую вы отправили боту.
- `{тип товара}` - тип товара строчными буквами для хэштега.
- `{тематика принта}` - тематический хэштег без знака `#`.

Старые поля `{title}`, `{description}`, `{size}`, `{price}` и `{hashtags}` также поддерживаются.

Стандартная подпись выглядит так:

```text
Футболка "Welcome to Turkmenistan"

Футболка, которая передает атмосферу нашей страны ❤️

👕 S-XXL

💸 250 манат

⏳ Готовность заказа: 1-2 дня

Чтобы заказать, отправьте:

✅ размер
✅ цвет
✅ номер телефона

📩 Заказы принимаем в сообщениях 👇

#Taýpa #футболка #туркменистан
```

Telegram разрешает до 1024 символов в подписи к фотографии. Бот проверяет это до публикации.

## Постоянная работа 24/7

Запланированные публикации отправляются только пока бот запущен. Для постоянной работы разместите проект на VPS с Docker:

```bash
cp .env.example .env
nano .env
docker compose up -d --build
```

Проверка журнала:

```bash
docker compose logs -f bot
```

Остановка:

```bash
docker compose down
```

Для бесплатного Render Web Service используйте файл `render.yaml` и инструкцию
`RENDER_SETUP.md`. Проект открывает страницу `/health` и одновременно запускает
Telegram polling. Для защиты от засыпания бесплатного сервиса настройте внешний
HTTP-монитор с интервалом 5 минут.

## V4.8 optional print PNG

- Исходное фото товара используется как основной источник принта.
- Отдельный PNG больше не требуется для запуска генерации.
- Кнопка генерации доступна сразу после анализа макета.


Если Gemini не может надежно измерить прямоугольные границы принта, бот больше не блокирует генерацию. Он анализирует изделие без координат и берет точный масштаб и положение принта непосредственно из исходного фото. Размер холста 2000x2000 не обязателен.


## Дополнительные администраторы

Основной ID остается в `ADMIN_TELEGRAM_ID`. Дополнительные ID укажите через запятую:

```env
ADMIN_TELEGRAM_IDS=987654321,555555555
```

Все указанные пользователи получают доступ к управлению ботом. Уведомления о публикации отправляются каждому администратору.

Кнопка `Написать` открывает чат `CONTACT_USERNAME` и вставляет только одну фразу: `Здравствуйте! Хочу заказать футболку «Название».`

## V5.2 динамический Pinterest и экономная маршрутизация

- После анализа товара бот сам составляет поисковые запросы Pinterest по изделию, полу модели, стороне принта, цвету, крою и настроению.
- Найденные Pins добавляются в PostgreSQL без дублей, скачиваются, проверяются и получают теги.
- При генерации сначала используются новые референсы, найденные именно под текущий товар. Если они не подошли, остается безопасный резерв из общей базы.
- Для Pinterest нужны `PINTEREST_ACCESS_TOKEN` и разрешение на `search/partner/pins`. Без них бот продолжит использовать уже готовую базу.
- Обычные и сложные задачи выполняет `flux-2-klein-4b`.
- Только очень сложные задачи выполняет `gemini-3.1-flash-lite-image`.
- Если `BFL_API_KEY` не заполнен, обычная задача останавливается без списания Gemini.
- `/check` показывает модель, сложность, состояние Pinterest и сколько ссылок найдено по последнему товару.
- Кнопка `Написать` вставляет только: `Здравствуйте! Хочу заказать футболку «Название».`
